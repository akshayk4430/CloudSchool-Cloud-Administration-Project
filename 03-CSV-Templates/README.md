# 📄 CSV Templates – CloudSchool Data Model

## 📌 Overview

This folder contains CSV files used as source data for user provisioning, group creation, and identity management.

The design follows a structured approach:

* CSV files act as the source of truth
* Identity data and business logic are separated
* Final provisioning uses validated and normalized datasets
* Group definitions are maintained separately from scripts

---

## 🎓 Student Data

### Files

```text
students.csv
students-template.csv
```

### Purpose

Student CSV files contain structured identity data for student provisioning.

Columns include:

* UserPrincipalName
* DisplayName
* GivenName
* Surname
* MailNickname
* Department
* EmployeeType
* UsageLocation
* ExtensionAttribute1
* ExtensionAttribute2

### Attribute Usage

```text
ExtensionAttribute1 = Grade
ExtensionAttribute2 = Division
```

Example:

```text
ExtensionAttribute1 = Grade01
ExtensionAttribute2 = DivisionA
```

Student data is already structured and does not require a merge step.

## 👨‍🏫 Staff Data

Staff provisioning uses a layered CSV model.

---

### 1. Identity Layer

File:

```text
staff.csv
```

Contains core identity data:

* UserPrincipalName
* DisplayName
* GivenName
* Surname
* MailNickname
* Department
* EmployeeType
* UsageLocation

This file represents the base staff identity dataset.

---

### 2. Attribute Mapping Layer

File:

```text
staff-attributes.csv
```

Contains business attributes:

* ExtensionAttribute1 → Role
* ExtensionAttribute2 → Assignment

Examples:

```text
Role: Principal, DeptHead, ClassTeacher, GradeCoordinator, Staff
Assignment: IT, Accounts, Operations, Grade01A
```

This keeps business logic separate from core identity data.

---

### 3. Final Provisioning Layer

File:

```text
staff-final.csv
```

Generated by merging:

* `staff.csv`
* `staff-attributes.csv`

Used by:

```text
03-create-staff.ps1
```

This file is the final staff provisioning source of truth.

---

## 👥 Group Data

### Group Source of Truth

File:

```text
groups-required.csv
```

Used by:

```text
04-Create-Groups.ps1
```

Purpose:

* Defines all required Entra ID groups
* Supports organizational, role-based, and service/policy groups
* Allows future group additions without rewriting the script

Columns:

* DisplayName
* MailNickname
* Description
* Category

Group categories:

```text
Organizational
Role
Service
Policy
```

Total managed groups:

```text
49
```

## ⚙️ Workflow Summary

### Student Workflow

1. Maintain `students.csv`
2. Run `02-create-students.ps1`
3. Validate output results

### Staff Workflow

1. Export staff data → `staff-only-export.csv`
2. Clean and normalize → `staff.csv`
3. Define roles and assignments → `staff-attributes.csv`
4. Merge datasets → `staff-final.csv`
5. Provision users using `03-create-staff.ps1`

### Group Workflow

1. Maintain required groups in `groups-required.csv`
2. Run `04-Create-Groups.ps1`
3. Validate `group-creation-results.csv`

---

## 📌 Key Principles

* CSV files are the source of truth
* Identity data is separated from business logic
* Group definitions are separated from group assignment logic
* Final datasets are normalized before provisioning
* Scripts are idempotent and safe to re-run
* Changes should be made in CSV files, not hardcoded directly into scripts

---

## ⚠️ Notes

* Always validate CSV data before running scripts
* Ensure no duplicate UserPrincipalNames
* Ensure no duplicate group DisplayNames
* Maintain consistent naming conventions
* Test with a small dataset before full execution
